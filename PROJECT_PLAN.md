# 農作物品質分類遊戲 - 專案計畫

## 計畫理解

### 教學目標
透過互動式遊戲，讓學員：
1. **親身體驗** 手動分類農作物的困難（速度快、數量多時難以應付）
2. **理解需求** 為什麼需要 AI 影像辨識來輔助
3. **實作訓練** 學習 YOLO-8 模型的標註與訓練流程
4. **整合應用** 將訓練好的模型整合到遊戲中實現自動分類

---

## 遊戲設計

### 核心玩法
```
[圖片來源] → [輸送帶] → [分類區]
                ↓
        玩家按按鈕推送
                ↓
    ┌─────────┬─────────┐
    │  健康區  │  病害區  │
    └─────────┴─────────┘
```

### 遊戲流程
1. 農作物圖片從輸送帶左側出現
2. 圖片隨輸送帶向右移動
3. 玩家觀察作物狀態（健康/病害）
4. 在作物到達分類點時，按下對應按鈕推送
5. 系統判斷分類是否正確並計分

### 難度遞增機制
| 階段 | 輸送帶速度 | 作物數量 | 間隔時間 |
|------|-----------|---------|---------|
| Level 1 | 慢 | 1個 | 長 |
| Level 2 | 中 | 2-3個 | 中 |
| Level 3 | 快 | 4-5個 | 短 |
| Level N | 極快 | 密集 | 極短 |

**設計目的**：讓學員發現當速度和數量增加時，人工分類變得不可能，自然引出 AI 輔助的必要性。

## 技術架構

### 專案結構
```
10_Quality_Selector_Game/
├── index.html              # 主頁面
├── css/
│   └── style.css           # 樣式
├── js/
│   ├── game.js             # 遊戲主邏輯
│   ├── conveyor.js         # 輸送帶控制
│   └── classifier.js       # 分類判斷邏輯
├── images/
│   ├── healthy/            # 健康作物圖片（使用者提供）
│   │   ├── apple_01.jpg
│   │   ├── apple_02.jpg
│   │   └── ...
│   └── diseased/           # 病害作物圖片（使用者提供）
│       ├── apple_01.jpg
│       ├── apple_02.jpg
│       └── ...
├── models/                  # 未來放置 YOLO 模型
│   └── yolov8_crop.onnx
└── PROJECT_PLAN.md         # 本計畫文件
```

### 技術選擇
- **前端框架**：純 HTML/CSS/JavaScript（方便教學，無需額外環境）
- **動畫效果**：CSS Animation + requestAnimationFrame
- **AI 整合**：ONNX.js 或 TensorFlow.js（在瀏覽器中運行模型）

## 圖片準備指南

### 建議規格
- **格式**：JPG 或 PNG
- **尺寸**：建議統一 200x200 或 300x300 像素
- **數量**：每類至少 20-30 張（遊戲用），訓練用則需更多

### 資料夾命名
```
images/
├── healthy/     # 健康作物
└── diseased/    # 病害作物（可細分不同病害類型）
    ├── bacterial/   # 細菌性病害
    ├── fungal/      # 真菌性病害
    └── pest/        # 蟲害
```
